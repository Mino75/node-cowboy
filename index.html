Cowboy

A pluggable parallel execution tool implemented with Node.js and Redis

Download this project as a .zip file Download this project as a tar.gz file

Cowboy is a light-weight, easy-to-install tool that lassoes a large number of servers together in order to perform execution tasks or gather diagnostic information from nodes in parallel. At its core, Cowboy simply facilitates a framework and infrastructure that allows plugins to be implemented that do useful things. We have a few helpful core plugins to get you started:

  • describe - Describes the modules and commands available to all listening cattle nodes
  • install - Installs a new cowboy module on all listening cattle nodes
  • ping - Interrogates the network for cattle nodes that are listening for commands
  • uninstall - Uninstalls an installed cowboy module on all listening cattle nodes

And the rest is up to you!

Installation

First download and install Redis, make sure it's listening on port 6379.

Then install the cowboy module:

npm -g install cowboy

Since Cowboy uses Bunyan for logging, it's best to install globally as well for piping log output:

npm -g install bunyan

Usage

Cowboy is made up of 2 components:

Cattle: The cattle binary listens to commands issued from the cowboy command VIA Redis. There must be a cattle process running on each server node that you control with Cowboy. This can be started by simply running: cattle | bunyan. For use in production, you will want to use something like Forever or Upstart to run it as a daemon process.

Cowboy: The cowboy module is the command-line client where commands can be executed. To see options, type: cowboy --help
